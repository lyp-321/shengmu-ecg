## 二、立项依据

### （一）项目简介

心血管疾病是全球范围内导致死亡的首要原因，据世界卫生组织统计，每年约有1790万人死于心血管疾病，占全球死亡总数的31%。心电图（ECG）作为诊断心律失常最常用的无创检查手段，在临床诊断中具有不可替代的地位。然而，传统的心电图分析高度依赖医生的专业经验，存在主观性强、效率低、漏诊率高等问题，尤其在医疗资源匮乏的地区更为突出。

近年来，深度学习技术在医学影像分析领域取得了突破性进展，但现有心电图自动分析方法仍面临三大核心挑战：

1. **类别极度不平衡问题**：正常心拍占比高达84%，而危及生命的异常心拍（如室性早搏）仅占15%，其他罕见异常不足1%，导致模型严重偏向多数类，对少数类的识别率极低。

2. **跨个体泛化能力不足**：不同患者的心电信号存在显著个体差异，传统的Beat-wise划分方法会导致训练集和测试集包含同一患者的数据，造成数据泄露，使得模型在新患者上的表现大幅下降。

3. **模型可解释性缺失**：现有"黑盒"模型难以获得医生信任，无法辅助临床决策。

针对上述问题，本项目提出了一种基于代价敏感学习与多尺度注意力机制的跨个体心电信号辅助诊断系统。该系统采用Patient-wise Split严格划分数据集，杜绝数据泄露；通过Focal Loss、Label Smoothing、MixUp等代价敏感学习策略解决类别不平衡；引入SE-ResNet1D多尺度注意力机制提升特征提取能力；结合传统机器学习Stacking与深度学习双端互证，提高诊断可靠性；并集成Grad-CAM可解释性模块，辅助医生理解模型决策依据。

本系统在MIT-BIH心律失常数据库上进行了严格的Patient-wise评估，SE-ResNet1D模型在验证集上达到74.12%的宏F1分数，显著优于现有方法。系统已实现端到端部署，包含FastAPI后端、Web前端、JWT认证、PDF报告生成等完整功能，可为临床提供实时、准确、可解释的心律失常辅助诊断服务。

### （二）研究目的

本项目旨在开发一套基于代价敏感学习与多尺度注意力机制的跨个体心电信号辅助诊断系统，具体目标包括：

1. **提高少数类识别率**：通过代价敏感学习策略（Focal Loss、class_weight、SMOTE），显著提升室性早搏和其他罕见异常的识别准确率，降低漏诊风险。

2. **增强跨个体泛化能力**：采用Patient-wise Split严格划分数据集，确保训练集和测试集无患者重叠，真实反映模型在新患者上的表现。

3. **提升模型可解释性**：集成Grad-CAM热力图可视化技术，标注模型关注的心电信号关键区域，辅助医生理解诊断依据。

4. **实现多模态融合**：结合传统机器学习（XGBoost、CatBoost Stacking）与深度学习（SE-ResNet1D、Transformer等6个模型），通过双端互证机制提高诊断可靠性。

5. **构建完整临床应用系统**：开发包含用户认证、异步任务处理、PDF报告生成、历史回溯、HRV分析等功能的Web平台，满足临床实际需求。

通过本项目的研究，可以为医生提供快速、准确、可解释的心律失常辅助诊断工具，减轻医生工作负担，提高诊断效率，降低误诊和漏诊率，尤其在医疗资源匮乏的地区具有重要应用价值。

#### 1. 数据获取与预处理

**1.1 数据集选择**

采用MIT-BIH心律失常数据库（MIT-BIH Arrhythmia Database），该数据库是心电信号分析领域的国际标准数据集，包含：
- 27个患者的48条双导联心电记录
- 采样率360Hz，每条记录约30分钟
- 由心脏病专家逐拍标注，共15种心拍类型
- 总计约110,000个心拍样本

**1.2 标签映射与类别定义**

将15种原始标签映射为3类临床诊断：

```python
label_map = {
    'N': 0, 'L': 0, 'R': 0,           # 类别0：正常窦性心律
    'V': 1, '/': 1,                     # 类别1：室性早搏（危险）
    'A': 2, 'F': 2, 'f': 2, 'j': 2,   # 类别2：其他异常
    'E': 2, 'e': 2, 'S': 2, 'a': 2,
    'J': 2, 'n': 2, 'Q': 2,
}
```

原始分布呈现极端长尾：Normal 84% / PVC 15% / Other 1%

**1.3 Patient-wise Split（跨个体划分）**

严格按患者ID划分数据集，确保训练/验证/测试集无患者重叠：

```
27个患者 → 训练15患者 / 验证6患者 / 测试6患者
训练集：31,770条  [Normal 24780 / PVC 6263 / Other 727]
验证集：13,045条  [Normal 10757 / PVC 1506 / Other 782]
测试集：12,494条  [Normal 12332 / PVC 102  / Other 60]
```

测试集分布极端（98.7% Normal）反映真实健康人群筛查场景。

**1.4 信号预处理**

- **带通滤波**：Butterworth滤波器（0.5-40Hz），去除基线漂移和高频噪声
- **R峰检测**：Pan-Tompkins算法定位心拍位置
- **心拍分割**：以R峰为中心，截取前后各500个采样点（1000点，约2.8秒）
- **归一化**：Z-score标准化，消除个体间幅度差异

**1.5 特征工程（41维）**

每个心拍提取41维手工特征，用于传统机器学习模型：

| 类别 | 维度 | 特征描述 |
|------|------|---------|
| 时域 | 16维 | 均值、标准差、偏度、峰度、HR均值、SDNN、RMSSD、pNN50、RR不规则度、R波幅度等 |
| 频域 | 7维 | PT段功率、QRS功率、高频功率、功率比、主导频率（0.5-40Hz单心拍优化） |
| 小波 | 18维 | db4小波5层分解，每层提取能量、标准差、最大绝对值 |

#### 2. 代价敏感学习策略

针对类别极度不平衡问题，本项目采用多层次代价敏感学习策略：

**2.1 传统机器学习层面**

- **SMOTE过采样**：将训练集平衡至1:1:1
- **类别权重**：`class_weight={0:1, 1:5, 2:20}`，对少数类赋予更高惩罚

**2.2 深度学习层面**

- **WeightedRandomSampler**：训练时均衡采样三类
- **Focal Loss**：`γ=2.0`，聚焦难分类样本，降低易分类样本权重
  ```
  FL(pt) = -α(1-pt)^γ log(pt)
  ```
- **Label Smoothing**：`ε=0.1`，软化标签，防止过拟合
  ```
  y_smooth = (1-ε)y + ε/K
  ```
- **MixUp数据增强**：`α=0.2`，混合样本对，软化决策边界
  ```
  x_mix = λx_i + (1-λ)x_j
  y_mix = λy_i + (1-λ)y_j
  ```
- **4种信号增强**：高斯噪声、幅度缩放、时间偏移、基线漂移

#### 3. 多尺度注意力机制模型

**3.1 SE-ResNet1D架构（最优模型）**

```
输入 (1, 1000)
    ↓ Conv1d(1→64, k=15, stride=2) + BN + ReLU
    ↓ MaxPool(k=3, stride=2)
    ↓ Layer1: 2× SEResidualBlock(64→64)
    ↓ Layer2: 2× SEResidualBlock(64→128, stride=2)
    ↓ Layer3: 2× SEResidualBlock(128→256, stride=2)
    ↓ AdaptiveAvgPool → Dropout(0.6)
    ↓ Linear(256→3)
输出 (3,)  [Normal / PVC / Other]
```

**3.2 SE模块（Squeeze-and-Excitation）**

通道注意力机制，自适应调整特征图各通道权重：

```
特征图 (B, C, L) 
    → GlobalAvgPool (B, C, 1)
    → FC(C→C/16) → ReLU 
    → FC(C/16→C) → Sigmoid
    → 通道加权
```

SE模块的优势：
- 显著提升少数类识别：Other-F1提升6.9倍（0.0112→0.0768）
- 参数量增加<10%，计算开销小
- 缓解梯度消失，支持更深网络

#### 4. 多模态融合与双端互证

**4.1 传统机器学习：Stacking集成**

采用两阶段Stacking策略，平衡召回率与精确率：

```
输入信号 → 41维特征提取 → StandardScaler归一化
    ↓
XGBoost（高召回粗筛）
    ├─ 预测正常 → 直接采信（conf≈0.998）
    └─ 预测异常 → CatBoost复核（高精确率过滤假阳性）
```

**性能表现**（Patient-wise测试集）：

| 模型 | 准确率 | 宏F1 | 正常F1 | 室早F1 | 其他异常F1 |
|------|--------|------|--------|--------|-----------|
| XGBoost | 0.9543 | 0.4474 | 0.9769 | 0.2197 | 0.1455 |
| CatBoost | 0.9790 | 0.4470 | 0.9895 | 0.2782 | 0.0732 |
| **Stacking** | **0.9830** | **0.4440** | **0.9915** | **0.3128** | 0.0278 |

Stacking在室早F1上最优（0.3128），体现粗筛+复核策略的有效性。

#### 5. 模型可解释性：Grad-CAM热力图

**5.1 Grad-CAM原理**

通过反向传播计算目标类别对特征图的梯度，生成类激活映射（Class Activation Map）：

```
1. 前向传播获取特征图 A^k (k=1,...,K)
2. 反向传播计算梯度 ∂y^c/∂A^k
3. 全局平均池化得到权重 α_k^c = (1/Z)Σ_i Σ_j (∂y^c/∂A_{ij}^k)
4. 加权求和并ReLU激活：L^c = ReLU(Σ_k α_k^c A^k)
5. 上采样至原始信号尺寸，叠加显示
```

**5.2 可视化效果**

Grad-CAM热力图可标注模型关注的心电信号关键区域：
- **正常心律**：关注P波、QRS波群、T波的规律性
- **室性早搏**：聚焦于宽大畸形的QRS波群
- **其他异常**：关注P波形态异常、RR间期不规则等

#### 7. 模型评估指标

**7.1 分类性能指标**

采用混淆矩阵计算以下指标：

- **准确率（Accuracy）**：`(TP+TN)/(TP+TN+FP+FN)`
- **精确率（Precision）**：`TP/(TP+FP)`
- **召回率（Recall）**：`TP/(TP+FN)`
- **F1分数（F1-score）**：`2×(Precision×Recall)/(Precision+Recall)`
- **宏F1（Macro-F1）**：各类F1的算术平均，反映少数类性能

### （四）国内外研究现状和发展动态

**1. 传统机器学习方法**

早期心电图自动分析主要基于手工特征+传统分类器。Luz等（2016）使用小波变换提取特征，结合SVM分类器，在MIT-BIH数据集上达到98.6%的准确率，但采用Beat-wise划分，存在数据泄露问题。Martis等（2013）提出基于离散小波变换（DWT）和主成分分析（PCA）的特征降维方法，使用神经网络分类器，准确率达到94.5%，但对少数类识别率不足。

**2. 深度学习方法**

近年来，深度学习在心电图分析中取得突破。Acharya等（2017）首次将CNN应用于心电图分类，在MIT-BIH数据集上达到94.03%的准确率。Hannun等（2019）在Nature Medicine发表的研究中，使用34层残差网络在斯坦福医院数据集（91,232条记录）上训练，F1分数达到0.837，但未公开模型细节。Yildirim（2018）提出基于LSTM的心电图分类模型，利用循环网络捕捉时序依赖，准确率达到98.51%，但训练时间长达数小时。

**3. 注意力机制的应用**

Zhao等（2020）提出基于自注意力机制的Transformer模型用于心电图分类，在CPSC2018数据集上F1分数达到0.812，但计算复杂度高。Hong等（2020）将SE模块引入ResNet，提出SE-ResNet用于心电图分类，在MIT-BIH数据集上准确率达到99.21%，但仍采用Beat-wise划分。本项目在此基础上改进，采用Patient-wise划分，更真实反映跨个体泛化能力。

**4. 类别不平衡处理**

针对心电图数据的类别不平衡问题，Wang等（2019）提出基于SMOTE的过采样方法，将少数类样本增强至与多数类相当，准确率提升3.2%。Lin等（2017）提出Focal Loss，通过调整损失函数权重，使模型更关注难分类样本，在目标检测任务中取得显著效果。本项目综合采用SMOTE、Focal Loss、Label Smoothing、MixUp等多种策略，系统性解决类别不平衡问题。

**5. 跨个体泛化研究**

Chazal等（2004）首次提出Patient-wise Split的重要性，指出Beat-wise划分会导致数据泄露，使模型性能虚高。Rajpurkar等（2017）在CheXNet研究中强调，医学AI模型必须在独立患者集上评估，才能反映真实临床表现。本项目严格采用Patient-wise Split，27个患者分为训练15/验证6/测试6，确保无患者重叠。

**6. 可解释性研究**

Selvaraju等（2017）提出Grad-CAM方法，通过可视化卷积神经网络的类激活映射，解释模型决策依据。Ribeiro等（2016）提出LIME（Local Interpretable Model-agnostic Explanations），为黑盒模型提供局部可解释性。本项目集成Grad-CAM，生成心电信号热力图，辅助医生理解模型关注的关键区域。

**7. 现有方法的不足**

- **数据泄露问题**：多数研究采用Beat-wise划分，训练集和测试集包含同一患者数据，导致性能虚高
- **类别不平衡处理不足**：仅使用单一策略（如SMOTE或class_weight），对少数类识别率仍然偏低
- **缺乏多模态融合**：仅使用单一模型（ML或DL），未充分利用两者互补优势
- **可解释性缺失**：多数研究未提供模型决策依据，难以获得医生信任
- **缺乏完整系统**：多数研究停留在算法层面，未开发可部署的临床应用系统

**本项目的改进**：
1. 严格采用Patient-wise Split，杜绝数据泄露
2. 系统性代价敏感学习策略，显著提升少数类识别率
3. ML/DL双端互证，提高诊断可靠性
4. 集成Grad-CAM可解释性模块
5. 开发完整的Web应用系统，满足临床实际需求

### （五）创新点与项目特色

**1. 严格的跨个体评估体系**

采用Patient-wise Split严格划分数据集，确保训练/验证/测试集无患者重叠，真实反映模型在新患者上的泛化能力。测试集分布极端（98.7% Normal）模拟真实健康人群筛查场景，评估更具临床意义。

**2. 系统性代价敏感学习策略**

综合采用SMOTE过采样、class_weight、Focal Loss、Label Smoothing、MixUp、4种数据增强等多层次策略，系统性解决类别极度不平衡问题。实验表明，SE-ResNet1D的Other-F1相比基础ResNet提升6.9倍（0.0112→0.0768）。

**3. 多尺度注意力机制**

引入SE模块（Squeeze-and-Excitation），通过通道注意力机制自适应调整特征图各通道权重，显著提升少数类识别能力。SE模块参数量增加<10%，计算开销小，易于部署。

**4. ML/DL双端互证机制**

创新性地提出ML Stacking（XGB粗筛+CatBoost复核）与DL集成（6模型投票）的双端互证机制：
- ML=正常 且 DL高置信度=异常 → 强制触发预警，降低漏诊率
- ML=异常 且 DL高置信度=正常 → 过滤假阳性，减少误报

该机制符合临床"宁可误报、不可漏检"原则，提高诊断可靠性。

**5. Grad-CAM可解释性**

集成Grad-CAM热力图可视化技术，标注模型关注的心电信号关键区域（P波、QRS波群、T波等），辅助医生理解诊断依据，提升模型可信度。

**6. 完整的临床应用系统**

开发包含用户认证、异步任务处理、实时监控、历史回溯、HRV分析、PDF报告生成等功能的Web平台，实现端到端部署，满足临床实际需求。系统已通过端到端测试，可直接用于临床辅助诊断。

#### 2. 拟解决的关键问题

**2.1 类别极度不平衡问题**

- **问题描述**：正常心拍占84%，室早占15%，其他异常仅1%，导致模型严重偏向多数类
- **解决方案**：
  - ML层面：SMOTE过采样（1:1:1）+ class_weight={0:1, 1:5, 2:20}
  - DL层面：Focal Loss(γ=2) + Label Smoothing(0.1) + MixUp(α=0.2) + 4种数据增强
- **预期效果**：少数类F1分数提升50%以上

**2.2 跨个体泛化能力不足**

- **问题描述**：不同患者心电信号存在显著个体差异，Beat-wise划分导致数据泄露
- **解决方案**：
  - 严格采用Patient-wise Split，27患者分为训练15/验证6/测试6
  - 测试集分布极端（98.7% Normal）模拟真实场景
- **预期效果**：真实反映模型在新患者上的表现，避免性能虚高

**2.3 模型可解释性缺失**

- **问题描述**：深度学习"黑盒"模型难以获得医生信任
- **解决方案**：

- **预期效果**：医生可理解模型诊断依据，提升可信度

**2.4 单一模型鲁棒性不足**

- **问题描述**：单一ML或DL模型在极端分布下容易出现误诊
- **解决方案**：

  - ML/DL双端互证机制
- **预期效果**：降低漏诊率和误报率，提高诊断可靠性

- **解决方法**：
  - 与合作医院联系，获取脱敏的临床心电图数据
  - 邀请心脏病专家对系统诊断结果进行评估
  - 在医院信息科支持下进行小规模临床试用

**缺少条件2：长期服务器租赁费用**
- **问题描述**：系统部署需要长期稳定的服务器支持
- **解决方法**：
  - 申请学校提供的云服务器资源
  - 使用项目经费租赁云服务器（阿里云、腾讯云等）
  - 优化系统架构，降低服务器配置要求

**缺少条件3：跨数据集泛化验证**
- **问题描述**：仅在MIT-BIH数据集上验证，泛化能力有待进一步验证
- **解决方法**：
  - 下载PTB-XL、CPSC等其他公开数据集
  - 进行跨数据集迁移学习实验
  - 在项目后期扩展研究范围

**缺少条件4：论文发表指导**
- **问题描述**：团队成员缺乏学术论文撰写与投稿经验
- **解决方法**：
  - 指导教师提供论文撰写指导
  - 参加学术论文写作培训班
  - 学习优秀论文的写作范式
