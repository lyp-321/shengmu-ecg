# 深度学习过拟合问题修复方案

## 问题诊断

训练结果显示严重过拟合：
- 训练准确率：99%+
- 验证准确率：50-60%
- Macro-F1：0.42-0.55

**根本原因：**
1. **Patient-wise split + 数据量不足**：27个患者，验证集只有5-6个，患者间ECG形态差异大
2. **随机采样损失数据**：原始10万条数据只用了6万条
3. **类别极度不均衡**：Normal:PVC:Other ≈ 84:15:1，Other类样本太少被重复采样
4. **模型容量过大**：ResNet 4层512通道对这个数据量来说太深

## 改进措施

### 1. 使用全量数据
```python
'num_samples': None,  # 从 60000 → None（全量约10万条）
```

### 2. 强化数据增强
在 `ECGDataset.__getitem__` 中添加：
- **高斯噪声**：SNR ~20dB，模拟真实采集噪声
- **幅度缩放**：[0.8, 1.2]，模拟不同导联增益
- **时间偏移**：±50采样点循环移位，增加时序鲁棒性
- **基线漂移**：低频正弦波，模拟呼吸/运动伪影

### 3. Label Smoothing
```python
'label_smoothing': 0.1  # 真实标签 = 0.9 * one_hot + 0.1/3
```
防止模型对训练集过度自信。

### 4. MixUp 数据增强
在 batch 层面混合样本：
```python
mixed_x = λ * x_i + (1-λ) * x_j
loss = λ * L(y_i) + (1-λ) * L(y_j)
```
软化决策边界，提升泛化能力。

### 5. 减小模型容量
- ResNet/SEResNet：4层 → 3层，512通道 → 256通道
- Dropout：0.5 → 0.6

### 6. 降低初始学习率
配合数据增强，避免早期震荡：
- ResNet/SEResNet/Inception：0.001 → 0.0005
- TCN：0.0005 → 0.0003
- BiLSTM/Transformer：0.0003 → 0.0002

## 预期效果

- **训练准确率**：降至 85-90%（正常，说明模型不再记忆训练集）
- **验证准确率**：提升至 70-75%
- **Macro-F1**：提升至 0.65-0.70
- **泛化能力**：验证集和测试集性能接近

## 使用方法

直接运行修改后的训练脚本：
```bash
python scripts/train_multimodal_models.py
```

所有改进已集成到代码中，无需额外配置。

## 进一步优化建议

如果效果仍不理想：
1. **增加验证集患者数**：改为 70/15/15 split
2. **使用更多数据集**：结合 PTB-XL、CPSC 等公开数据集
3. **迁移学习**：用 ImageNet 预训练的 ResNet 初始化
4. **集成学习**：多模型投票或 stacking
